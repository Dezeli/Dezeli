# 📝 Commit Message Convention
하나의 커밋에는 하나의 이슈만 들어간다.

## 💡 기본 구조
```text
[Type] : 커밋의 주제


 - 상세 내용 수정
 - 상세 내용 삭제
```

## 📋 작성 규칙
1. **Type**: 첫 글자는 반드시 **대문자**로 작성한다.
2. **Spacing**: Title(주제) 아래에는 반드시 **빈 줄 두 개(Enter 2번)**를 삽입한다.
3. **List**: `-`을 쓸 때는 앞에 **공백 한 칸**을 스페이스로 들여쓴다.
4. **Content**: 무엇을 했는지 **간단 명료**하게 쓴다.

---

## 🏷️ Type 종류 및 사용 시점

| Type | 사용 시점 |
| :--- | :--- |
| **Feat** | 새로운 기능 추가 |
| **Fix** | 버그 수정 |
| **Docs** | 문서 수정 |
| **Style** | 코드 스타일 변경 (코드 포매팅, 세미콜론 누락 등) 기능 수정이 없는 경우 |
| **Design** | 사용자 UI 디자인 변경 (CSS 등) |
| **Test** | 테스트 코드, 리팩토링 테스트 코드 추가 |
| **Refactor** | 코드 리팩토링 |
| **Build** | 빌드 파일 수정 |
| **Ci** | CI 설정 파일 수정 |
| **Perf** | 성능 개선 |
| **Chore** | 빌드 업무 수정, 패키지 매니저 수정 (gitignore 수정 등) |
| **Rename** | 파일 혹은 폴더명을 수정만 한 경우 |
| **Remove** | 파일을 삭제만 한 경우 |
